# Contributing

## Git-workflow

- ветка `main` защищена, прямые коммиты запрещены;
- на каждую задачу — отдельная feature-ветка:
  - `feature/<что-делаем>` — новая функциональность;
  - `fix/<что-чиним>` — исправление;
  - `docs/<что-пишем>` — документация;
- каждое изменение проходит через Pull Request;
- после мёржа ветка удаляется.

## Commit-сообщения

Используем **Conventional Commits**:

```
feat: добавить эндпоинт GET /places
fix: исправить фильтр по бюджету
docs: написать PRD
test: добавить API-автотесты для отзывов
```

## Definition of Done

- код покрыт тестами;
- проходят Ruff, Black, mypy;
- обновлена документация, если менялось поведение;
- зелёный CI;
- ревью пройдено.
