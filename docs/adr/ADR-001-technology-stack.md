# ADR-001: Технологический стек

- **Статус:** Accepted
- **Дата:** 2026-07-20
- **Автор:** Adel Gabdullin

## Контекст
Нужен стек для коммерческого backend-продукта с REST API, асинхронной обработкой и наблюдаемостью, который также служит инженерным портфолио-кейсом (QA Automation / Python Backend).

## Решение
Backend: **Python 3.13 + FastAPI + SQLAlchemy + Alembic**. Хранилище: **PostgreSQL**. Кэш: **Redis**. Очередь: **RabbitMQ**. Контейнеризация: **Docker + Docker Compose**. Frontend: **React + TypeScript**. Наблюдаемость: **Prometheus + Grafana**. CI/CD: **GitHub Actions**.

## Рассмотренные альтернативы
- Django — тяжелее и монолитнее для API-first продукта, менее удобная работа с async и OpenAPI.
- Flask — минималистичен, но требует больше ручной сборки (валидация, OpenAPI, DI).

## Последствия
Плюсы: нативная OpenAPI-документация, async, строгая валидация через Pydantic, современный стек, хорошо смотрится на собеседовании. Минусы: широкий стек увеличивает объём работ по инфраструктуре и DevOps.
