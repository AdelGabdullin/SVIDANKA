# ADR-002: Выбор базы данных

- **Статус:** Accepted
- **Дата:** 2026-07-20
- **Автор:** Adel Gabdullin

## Контекст
Нужны реляционные данные (места, города, категории, отзывы, модерация), геозапросы по расстоянию и надёжные связи между сущностями.

## Решение
**PostgreSQL** как основная БД, с возможностью подключения расширения **PostGIS** для гео-запросов на следующих этапах.

## Рассмотренные альтернативы
- MySQL — слабее по гео-возможностям и работе со сложными запросами.
- MongoDB — данные сильно связаны реляционно, документная модель избыточна и усложнит целостность.

## Последствия
Плюсы: строгая целостность, зрелая экосистема, PostGIS для карты/расстояний. Минусы: требуется управление миграциями (Alembic) и продуманная схема.
