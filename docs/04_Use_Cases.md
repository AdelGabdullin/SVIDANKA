## 2. UC-01. Подбор места по фильтрам

- **Актор:** Гость
- **Предусловие:** выбран город.
- **Триггер:** пользователь задаёт или меняет фильтры (бюджет, район, расстояние, категория).

**Основной поток:**

1. Пользователь открывает список и карту мест выбранного города.
2. Пользователь задаёт один или несколько фильтров.
3. Система применяет фильтры (комбинируя их по И) и обновляет список и карту.
4. Пользователь просматривает подходящие места и открывает карточку понравившегося (см. карточку места, `02_PRD.md` FR-5).

**Альтернативные потоки:**

- A1. Пользователь снимает часть фильтров — список и карта расширяются автоматически.

**Исключения:**

- E1. Под заданные фильтры не подходит ни одно место — система показывает сообщение об отсутствии подходящих мест (не ошибку).
- E2. Введено некорректное значение бюджета (отрицательное или больше 100 000 ₽) — значение не принимается, пользователю предлагается ввести корректное.

**Постусловие:** пользователь видит отфильтрованный список мест, готовый для дальнейшего выбора.

**Связанные требования:** FR-7, FR-8, FR-9 (`02_PRD.md`); US-04, US-05 (`03_User_Stories.md`).

## 3. UC-02. Случайный выбор (Кубик)

- **Актор:** Гость
- **Предусловие:** выбран город; фильтры заданы опционально.
- **Триггер:** нажатие кнопки «Кубик».

**Основной поток:**

1. Пользователь нажимает «Кубик».
2. Система выбирает случайное место среди мест, соответствующих текущим фильтрам.
3. Система открывает карточку выбранного места.

**Альтернативные потоки:**

- A1. Пользователю не подошло предложенное место — он повторно нажимает «Кубик», процесс повторяется.

**Исключения:**

- E1. Ни одно место не подходит под заданные фильтры — система показывает сообщение об отсутствии подходящих мест без ошибки.

**Постусловие:** пользователю показано одно случайное подходящее место либо сообщение об отсутствии таких мест.

**Связанные требования:** FR-10, FR-11; US-06, US-07.

## 4. UC-03. Добавление анонимного отзыва

- **Актор:** Гость
- **Предусловие:** пользователь открыл карточку места.
- **Триггер:** пользователь нажимает «Оставить отзыв».

**Основной поток:**

1. Пользователь указывает оценку (1–5) и пишет текстовый комментарий.
2. Пользователь отправляет форму.
3. Система сохраняет отзыв и сразу публикует его в карточке места (постмодерация).
4. Система пересчитывает средний рейтинг места.

**Альтернативные потоки:** —

**Исключения:**

- E1. Не указана оценка или нарушены ограничения по длине комментария — форма не отправляется, показывается ошибка валидации.

**Постусловие:** отзыв виден всем пользователям в карточке места; рейтинг места обновлён.

**Связанные требования:** FR-12, FR-13, FR-15, FR-16; US-08, US-09.

## 5. UC-04. Предложение нового места

- **Актор:** Гость
- **Предусловие:** пользователь не нашёл нужное место в базе.
- **Триггер:** пользователь нажимает «Предложить место».

**Основной поток:**

1. Пользователь заполняет форму: название, адрес, город, категория, бюджет, описание, фотографии.
2. Пользователь отправляет форму.
3. Система сохраняет место и сразу публикует его в общем каталоге (постмодерация).

**Альтернативные потоки:** —

**Исключения:**

- E1. Не заполнены обязательные поля или бюджет вне диапазона 0–100 000 ₽ — форма не отправляется, показывается ошибка валидации.

**Постусловие:** новое место видно всем пользователям в каталоге города.

**Связанные требования:** FR-17, FR-18; US-10.

## 6. UC-05. Модерация места

> Название сохранено из каркаса документа, но описывает **постфактум-действие**: в MVP принята постмодерация (см. ADR-004, `02_PRD.md` раздел 6.1) — место или отзыв публикуются сразу при создании, без предварительной проверки. Этот use case описывает, как администратор убирает уже опубликованный неподходящий контент.

- **Актор:** Администратор
- **Предусловие:** в каталоге есть место или отзыв, не соответствующие требованиям сервиса (не по теме свиданий/отдыха, оскорбления, спам, недостоверная информация).
- **Триггер:** администратор обнаруживает неподходящий контент (самостоятельно или по сигналу от пользователя).

**Основной поток:**

1. Администратор открывает админ-панель и просматривает места/отзывы.
2. Администратор находит место или отзыв, не соответствующие требованиям.
3. Администратор скрывает или удаляет этот контент.
4. Система прекращает показ этого контента всем пользователям.

**Альтернативные потоки:**

- A1. Администратор восстанавливает ранее скрытое место/отзыв, если решение было ошибочным.

**Исключения:** —

**Постусловие:** неподходящий контент больше не отображается в каталоге или карточке места.

**Связанные требования:** FR-20; US-11.

## 7. UC-06. Прямое добавление места администратором

- **Актор:** Администратор
- **Предусловие:** администратор авторизован в админ-панели.
- **Триггер:** администратор хочет добавить проверенное место, минуя форму предложения.

**Основной поток:**

1. Администратор открывает форму добавления места в админ-панели.
2. Администратор заполняет данные места: название, адрес, город, категория, бюджет, описание, фотографии.
3. Администратор сохраняет место.
4. Система публикует место в общем каталоге как обычное место, наравне с местами, добавленными через форму предложения.

**Альтернативные потоки:** —

**Исключения:**

- E1. Не заполнены обязательные поля — форма не сохраняется, показывается ошибка валидации.

**Постусловие:** место опубликовано в каталоге и доступно всем пользователям.

**Связанные требования:** FR-19; US-12.

## 8. UC-07. Управление справочниками (города и категории)

- **Актор:** Администратор
- **Предусловие:** администратор авторизован в админ-панели.
- **Триггер:** администратор хочет добавить или изменить город либо категорию.

**Основной поток:**

1. Администратор открывает раздел управления городами/категориями.
2. Администратор добавляет новый город/категорию или редактирует существующие.
3. Система сохраняет изменения.
4. Новый город/категория сразу становятся доступны в фильтрах, формах предложения места и карточках.

**Альтернативные потоки:**

- A1. Администратор деактивирует город или категорию, которые больше не нужны (поведение для уже привязанных к ним мест уточняется в `05_Business_Rules.md`/`06_Domain_Model.md`).

**Исключения:**

- E1. Попытка создать дублирующий город/категорию — система показывает ошибку.

**Постусловие:** справочник городов/категорий обновлён и используется во всех фичах сервиса без изменения кода.

**Связанные требования:** FR-2, FR-6, FR-21; US-13.
